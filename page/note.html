快速复制文件名

第一步：打开需要批量复制文件名称的文件夹；

第二步：在该文件夹下新建一个txt文件，然后将.txt后缀名修改为.bat，此时直接忽略系统弹出的警告；

第三步：右键打开刚刚改名的.bat文件，选择使用文本编辑器（推荐Notepad++）打开，复制如下内容到文件中，然后保存：

DIR *.* /B >LIST.TXT

第四步：双击.bat文件，即可在此文件夹下直接生成名字为LIST.TXT的文件名；

批量更改图片名称

第一步：将需要重新命名的图片放在一个文件夹内；

第二步：在该文件夹下新建一个txt文件，
ren 33.png reactions-carboxylic-acids-33.png
ren 33 --- 原文件名 .png --- 文件类型
reactions-carboxylic-acids-33.png (更改后的名称)

第三步：然后将.txt后缀名修改为.bat，此时直接忽略系统弹出的警告；

第四步：双击.bat文件运行 --- 文件有“（）” 符号不支持

Sublime Text 常用快捷键

选择类

  • Alt + F3选择全部的相同文本
  • Ctrl + D依次选中光标所占的文本
  • Ctrl + L依次向下选中行
  • Ctrl + Shift + L先选中多行，再按快捷键，会在每行行尾插入光标
  • Ctrl + M光标移动至 括号内 结束或开始的位置
  • Ctrl + Shift + M选择括号内的内容
  • Ctrl + Shift + Enter在上一行插入新行
  • Ctrl + Enter在下一行插入新行
  • Ctrl + ←/→向左/右移动光标
  • Ctrl + Shift + ↑/↓将光标所在行和上/下一行代码互换
  • Shift + ↑/↓/←/→向上/下/左/右选中多行

编辑类

  • Ctrl + J选择全部的相同文本
  • Ctrl + Shift + D复制光标所在整行，插入到行尾
  • Ctrl + K + K从光标处开始删除代码至行尾
  • Ctrl + Shift + K删除整行
  • Ctrl + F2设置书签
  • Ctrl + T左右字母互换
  • F6单词检测拼写

显示类

  • Alt + Shift + N 等分 N 屏( N 字母上方的数字)
  • Ctrl + K + B开启/关闭侧边栏
  • Shift + F11免打扰模式
  • F11 全屏模式

git 相关

git commit 时出现：please enter the commit message for your changes

可以看到，默认的提交消息包含最后一次运行 git status 的输出，放在注释行里，另外开头还有一空行，供你输入提交说明。你完全可以去掉这些注释行，不过留着也没关系，多少能帮你回想起这次更新的内容有哪些。（如果觉得这还不够，可以用 -v 选项将修改差异的每一行都包含到注释中来。）退出编辑器时，Git 会丢掉注释行，将说明内容和本次更新提交到仓库。

如果要输入解释的话就需要:

1.按键盘字母 i 进入insert模式

2.修改最上面那行黄色合并信息,可以不修改

3.按键盘左上角"Esc"

4.输入":wq",注意是冒号+wq,按回车键即可

js 相关

javascript 错误类型

Error 是基类型，其他错误类型都是继承该类型，浏览器很少会抛出 Error 类型的错误

InternalError 类型的错误会在底层 javascript 引擎抛出异常时由浏览器抛出

EvalError 类型错误会在使用 eval() 函数发生异常时抛出

RangeError 会在数值越界时抛出

ReferenceError 会在找不到对象时发生

SyntaxError 经常在给eval()传入的字符串包含 javascript 语法错误时发生

TypeError 很常见，主要发生变量不是预期类型，或者访问不存在的方法时等原因导致，尤其是在使用类型特定的操作而变量类型不对时

URIError 只会在使用 encodeURL() 或 decodeURL() 但传入了格式错误的 URL 时发生

同一个监听器注册多个处理器 (addEventListener,removeEventListener)

myElement.onclick = functionA;
myElement.onclick = functionB;
第二行会覆盖第一行，但是下面这种方式就会正常工作了：
myElement.addEventListener('click', functionA);
myElement.addEventListener('click', functionB);

当您要在多个元素上设置相同的事件处理程序时，e.target非常有用，并且在发生事件时对所有元素执行某些操作。

media查询 展示不同分辨率的图片

.avatar{ background-image: url(xx_1x.png); }

@media only screen and (-webkit-min-device-pixel-ratio:2){ .avatar{ background-image: url(xx_2x.png); } }

@media only screen and (-webkit-min-device-pixel-ratio:3){ .avatar{ background-image: url(xx_3x.png); } }
